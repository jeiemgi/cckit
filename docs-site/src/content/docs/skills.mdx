The skills cckit installs, grouped by what you're trying to do.

A **skill** is a packaged instruction set Claude Code can run. Every skill here you can run with a
`/` (for example `/kit-next`) — that's the <Tag kind="skill" text>Skill</Tag> tag. Some also trigger
when you just **describe the task in plain words** — that's the
<Tag kind="conversational" text>Conversational</Tag> tag. New to the tags? See
[How to read this guide](/how-to-read/).

## Everyday loop

<div class="cckit-nowrap-2">

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-next` | <Tag kind="skill" /> | Pick up the next unblocked issue, or fan the wave out. |
| `kit-ship` | <Tag kind="skill" /> | One request, code to merged. |
| `kit-autopilot` | <Tag kind="skill" /> | Run a planned effort hands-off. |
| `kit-task-sync` | <Tag kind="skill" /> | Print the board by role; flag blockers. |

</div>

## Task lifecycle (one issue)

<div class="cckit-nowrap-2">

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-task-new` | <Tag kind="skill" /> | Create an issue with labels and board fields. |
| `kit-task-start` | <Tag kind="skill" /> | Branch (optional worktree); mark it in progress. |
| `kit-task-pr` | <Tag kind="skill" /> | Commit, push, open a PR linked to the issue. |
| `kit-task-pr-merge` | <Tag kind="skill" /> | Squash-merge the PR, switch to base, pull. |
| `kit-task-pr-auto` | <Tag kind="skill" /> | Open the PR and merge it in one step. |
| `kit-task-close` | <Tag kind="skill" /> | Close the issue; set the board to Done. |

</div>

## Effort lifecycle (many issues, one goal)

<div class="cckit-nowrap-2">

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-effort-new` | <Tag kind="skill" /> | Create an effort: a parent plus sub-issues. |
| `kit-effort-start` | <Tag kind="skill" /> | Open the effort branch the subs merge into. |
| `kit-effort-pr` | <Tag kind="skill" /> | Open the one PR for the whole effort. |
| `kit-effort-close` | <Tag kind="skill" /> | Squash-merge subs, close parent + subs, clean up. |

</div>

## Maintenance & delivery

<div class="cckit-nowrap-2">

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-gc` | <Tag kind="skill" /> | Prune merged branches and worktrees. |
| `kit-security-sweep` | <Tag kind="skill" /> | Audit for injection, auth, CORS/CSP, deps. |
| `kit-digest` | <Tag kind="skill" /> | Pre-digest long inputs with a local model. |

</div>

## Authoring & contributing

<div class="cckit-nowrap-2">

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `docs-writer` | <Tag kind="conversational" /> | Write docs in the house voice. |
| `kit-customize` | <Tag kind="conversational" /> | Add or edit agents and skills in a project. |
| `kit-contribute` | <Tag kind="conversational" /> | Open a PR to contribute your additions upstream. |
| `kit-onboard` | <Tag kind="skill" /> | First-contact setup on a new machine. |

</div>

<Aside type="note" title="Running a skill">
In Claude Code, type `/` and the skill name (for example `/kit-next`). A
<Tag kind="conversational" text>conversational</Tag> skill also triggers when you describe the task —
for instance, asking to "contribute this upstream" starts `kit-contribute`.
</Aside>
