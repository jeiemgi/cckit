The skills cckit installs, grouped by what you're trying to do.

A **skill** is a packaged instruction set Claude Code can run. Every skill here you can run with a
`/` (for example `/kit-next`) — that's the <Tag kind="skill" /> tag. Some also trigger when you just
**describe the task in plain words** — that's the <Tag kind="conversational" /> tag. New to the
tags? See [How to read this guide](/how-to-read/).

## Everyday loop

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-next` | <Tag kind="skill" /> | Find the next unblocked issue and pick it up, or fan the whole wave out. |
| `kit-ship` | <Tag kind="skill" /> | Take one request from code to merged: start → PR → CI → merge → close. |
| `kit-autopilot` | <Tag kind="skill" /> | Run a planned effort hands-off — the captain merges each wave and continues. |
| `kit-task-sync` | <Tag kind="skill" /> | Print the board grouped by role; flag blocked items. |

## Task lifecycle (one issue)

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-task-new` | <Tag kind="skill" /> | Create an issue with labels, milestone, and board fields in one flow. |
| `kit-task-start` | <Tag kind="skill" /> | Branch (optionally an isolated worktree) and mark the issue in progress. |
| `kit-task-pr` | <Tag kind="skill" /> | Commit, push, and open a pull request linked to the issue. |
| `kit-task-pr-merge` | <Tag kind="skill" /> | Squash-merge the open PR, switch to the base branch, pull. |
| `kit-task-pr-auto` | <Tag kind="skill" /> | Open the PR and merge it in one command. |
| `kit-task-close` | <Tag kind="skill" /> | Close the issue and set the board to Done after its PR merges. |

## Effort lifecycle (many issues, one goal)

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-effort-new` | <Tag kind="skill" /> | Create an effort: a parent issue plus linked sub-issues on the board. |
| `kit-effort-start` | <Tag kind="skill" /> | Begin building — open the single effort branch the subs merge into. |
| `kit-effort-pr` | <Tag kind="skill" /> | Open the one pull request for the whole effort. |
| `kit-effort-close` | <Tag kind="skill" /> | Snapshot each sub's diff, squash-merge, close parent + subs, garbage-collect. |

## Maintenance & delivery

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `kit-gc` | <Tag kind="skill" /> | Prune merged branches and worktrees; surface orphans and stale stashes. |
| `kit-security-sweep` | <Tag kind="skill" /> | Audit the apps for injection, auth, CORS/CSP, and dependency issues. |
| `kit-digest` | <Tag kind="skill" /> | Pre-digest long inputs with a local model to save tokens. |

## Authoring & contributing

| Skill | Activates | What it does |
| ----- | --------- | ------------ |
| `docs-writer` | <Tag kind="conversational" /> | Write docs in the house voice — kind but robust, scannable, no analogies. |
| `kit-customize` | <Tag kind="conversational" /> | Add or edit agents and skills in a scaffolded project. |
| `kit-contribute` | <Tag kind="conversational" /> | Open a pull request to contribute your additions back to cckit. |
| `kit-annotate` | <Tag kind="conversational" /> | Wire click-to-annotate visual UI feedback for a React app. |
| `kit-onboard` | <Tag kind="skill" /> | First-contact setup on a new machine or project. |

<Aside type="note" title="Running a skill">
In Claude Code, type `/` and the skill name (for example `/kit-next`). A <Tag kind="conversational" />
skill also triggers when you describe the task — for instance, asking to "contribute this upstream"
starts `kit-contribute`.
</Aside>
