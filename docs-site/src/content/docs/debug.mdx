Give an agent a token-efficient, scriptable way to debug a web page — screenshots, accessibility snapshots, console and network logs, [Lighthouse](https://developer.chrome.com/docs/lighthouse/overview) — without bloating its context.

`cckit debug` drives [chrome-devtools-axi](https://github.com/kunchenguid/chrome-devtools-axi) (Kun Chen), which runs a self-contained [Chrome DevTools](https://developer.chrome.com/docs/devtools/) bridge and emits [TOON](/agents/#the-verbs-an-agent-uses-most)-encoded output. TOON is a compact text format, so results stay small in an agent's context.

<Aside type="tip" title="Built on chrome-devtools-axi">
The browser layer is **[chrome-devtools-axi](https://github.com/kunchenguid/chrome-devtools-axi)** by Kun Chen — a self-contained Chrome bridge for agents. cckit only wraps and auto-detects it; all credit for the browser tooling goes there. If it helps you, ⭐ the repo.
</Aside>

<Aside type="note" title="New here? Why this exists">
Debugging a web page normally means a person clicking around a browser. An agent cannot do that. `cckit debug` exposes the same browser tools as plain commands an agent can run and read, so the agent can see what is on the page.
</Aside>

## Optional + auto-detected

cckit is pure bash; axi needs Node and Chrome. So this is **not a hard dependency** — cckit detects it at call time and degrades cleanly:

- **Node/npx and Chrome present** → `cckit debug` drives axi.
- **Otherwise** → it prints how to enable the capability and exits `0`. An unattended run is never hard-failed by a missing optional tool.

Check availability without running anything:

```bash
cckit debug --check
```

## Enabling it

```bash
npm i -g chrome-devtools-axi      # or rely on `npx -y chrome-devtools-axi` (Node present)
```

Override the command cckit invokes with an environment variable — useful for a pinned version or a wrapper:

```bash
export CCKIT_AXI=<command>
```

## Usage

Everything after `cckit debug` is forwarded verbatim to axi, so cckit never has to track axi's evolving subcommand surface:

```bash
cckit debug <axi-args...>         # passed straight through — see the axi docs for subcommands
```
