## Why cckit

<CardGrid stagger>
  <Card title="One lifecycle" icon="seti:todo">
    Start an issue, get an isolated worktree and branch, open a PR, merge, clean up — each step is one command, correct by construction.
  </Card>
  <Card title="Agent-operable" icon="puzzle">
    Every verb has a machine-readable mode (`cckit <verb> --llm`) and an `AGENTS.md` contract, so agents drive the kit without scraping human output.
  </Card>
  <Card title="Zero hard dependencies" icon="approve-check">
    Pure bash with graceful fallbacks; it auto-detects optional tools and degrades instead of failing.
  </Card>
  <Card title="Portable" icon="setting">
    No company, framework, or repo is baked in — everything is driven from `cckit.config.json`.
  </Card>
</CardGrid>
