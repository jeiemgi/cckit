kit-annotate — implementation plan

What was added to claude-kit to wire Agentation into Claude, how it flows, what's verified, and what's next. Read 01-explainer.html first for the concepts.

ℹ

Decision (verified, unanimous in an 18-agent design pass): the Hybrid — ship a claude-kit scaffolder now whose default backend is Agentation, behind a named-backend seam so a kit-owned backend can slot in later if richer capture (live prop/state values) is ever needed. Scope: Claude Code only.

1Goal & decision

Goal: in-app, click-to-annotate UI feedback for React projects, wired into Claude via claude-kit — so the kit's job is the wiring, not the capture engine. Why adopt Agentation: it's source-available + standard MCP (passes the "inspectable" bar onUI failed), ~3.4M downloads/mo, and already an in-app wrapper across Next/Vite/SSR. License: PolyForm Shield 1.0.0 — a noncompete; installing it as the user's own dev dependency is fully permitted; the kit must not vendor its code.

⚠

Capability boundary, verified against Agentation's source: it captures component-name tree + selectors + computed styles + a source path — not live prop/state values. That sanitized props/state snapshot is the one thing only a kit-owned backend could add; it stays parked until the need is validated on a real PWA.

3Setup & session flow

Setup (/kit-annotate setup)

  1. Dry-run the engine → show framework, entry file, and plan.
  2. Explain capabilities + honest limits + license; ask consent (Install / Preview / Skip).
  3. Engine: npm i -D agentation → claude mcp add agentation -- npx -y agentation-mcp server → write .claude/rules/react-annotate.md → merge .annotate into kit.config.json.
  4. Wire <Agentation /> into the entry via Edit (diff shown, dev-gated), per framework:
    FrameworkEntryGate
    Next.js (App)app/layout.tsx (client)process.env.NODE_ENV === "development"
    Next.js (Pages)pages/_app.tsxsame
    Vitesrc/main.tsximport.meta.env.DEV
    React Router v7app/root.tsximport.meta.env.DEV
  5. Restart Claude Code (loads the MCP) → run dev server → npx agentation-mcp doctor.

Session (/kit-annotate watch / review)

The loop, also encoded in the rule: pull (agentation_get_all_pending or hands-free agentation_watch_annotations) → agentation_acknowledge → locate (source pointer or grep selector/component) → fix → agentation_resolve with a summary (or agentation_dismiss / agentation_reply). On close, optional MemPalace summary to the project wing.

4Config & data flow

The choice persists in .claude/kit.config.json under a named-backend block (so a future kit-owned backend slots in without a schema break):

"annotate": {
  "enabled": true,
  "backend": "agentation",          // ← named, swappable
  "framework": "next",
  "reactVersion": "19.2.0",
  "entryFile": "app/layout.tsx",
  "package": "agentation",
  "mcp": { "name": "agentation", "server": "agentation-mcp", "httpPort": 4747 }
}
